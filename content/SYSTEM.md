# Content System

## Goal

One topic per week. Each topic produces:
- 1 LinkedIn post (200–300 words, opinion-led, personal voice)
- 1 dev.to article (800–1500 words, storytelling + code + opinion)
- 1 GitHub commit (working notebook, linked from both)

## Content Identity

> "I've been shipping AI products with Claude. Now I'm going back to the foundations — and I have opinions about what I find."

**Series name:** Building with Claude from the inside out

## Publishing Order

1. **Commit first** (timestamp matters)
2. **dev.to article** — links to the notebook
3. **LinkedIn post** — links to the dev.to article

## Tone Rules

- Write in first person always
- Start with a moment, a question, or a provocation — never a definition
- One concrete opinion per post, stated plainly
- Show the code, show what surprised you, show what you'd do differently
- End LinkedIn posts with a question to drive comments
- End dev.to articles with "what's next" and a repo link
